Project Documentation

Isaac's Online Voting System

A secure, transparent, and accessible digital platform designed to modernize student elections for KE numbers 001–550.

📋 Executive Summary

This project empowers eligible students by providing a simple, intuitive voting experience while delivering real-time transparency for administrators. By digitizing the workflow, we address critical pain points of traditional paper-based elections: speed, accuracy, accessibility, and trust.

👥 Helping Voters & Students

📱 Accessibility

Cast ballots from any device—mobile, tablet, or desktop. No need to queue physically.

🛡️ Integrity

Prevents accidental duplicate voting by locking each KE number after submission.

🧭 Guidance

Step-by-step interface with visual progress tracking and clear candidate cards.

User Experience Highlights

  • Intuitive Interface: Guides voters through all 10 leadership positions seamlessly.
  • Instant Confirmation: Users know immediately that their vote counts.
  • Educational Guide: A dedicated "How It Works" page reduces confusion with FAQs.

🔒 Helping Election Administrators

Real-Time Transparency

The password-protected admin dashboard automatically aggregates votes as they are submitted. Officials can view live results with visual progress bars, eliminating hours of manual counting and reducing human error.

🖨️ Print-Ready Reports

Generate professional, official-looking result reports with a single click using optimized CSS print styles.

🔄 Easy Management

"Reset All" feature provides a clean slate for testing. Data persists via localStorage without complex servers.

⚙️ Improving the Overall Election Process

By digitizing the voting workflow, the project addresses critical pain points of traditional paper-based elections.

Key Improvements

  • Speed & Accuracy: Votes are recorded instantly and securely in the browser's localStorage with structured keys (election_votes_2026).
  • Eligibility Enforcement: KE number validation (3-digit, range 001–550) ensures only eligible students vote.
  • Inclusivity: Responsive, mobile-first design ensures students with limited computer access can participate via smartphones.
  • Trust: Separation of voter and admin interfaces builds confidence in the system's fairness.

❓ Why This Project Was Created

Traditional paper-based voting systems often face challenges such as manual counting errors, delayed results, duplicate voting, and limited accessibility.

This Online Voting System was developed to modernize and streamline student elections by providing a secure, transparent, and accessible digital platform. It enables eligible students to cast votes electronically while ensuring one-vote-per-user integrity, promoting trust and efficiency in the electoral process.

🛠️ How The Project Was Built

Architecture Overview

The application uses a client-side architecture with four interconnected HTML pages:

  1. index.html: Landing page with navigation.
  2. voter.html: Secure voting interface with identity verification.
  3. admin.html: Password-protected results dashboard.
  4. guide.html: User documentation and FAQs.

The Voting Flow

  • Verification: Identity check via 3-digit KE number.
  • Selection: Step-by-step candidate selection across 10 positions.
  • Storage: Selections stored temporarily in a JS object, then aggregated to localStorage upon submission.
  • Rendering: Admin panel reads localStorage to render live visual progress bars.

Key Interactions

Handled entirely with Vanilla JavaScript:

  • Form validation (KE Number range 001-550)
  • Dynamic candidate rendering
  • Progress tracking & Modal dialogs
  • Print functionality for official reports
